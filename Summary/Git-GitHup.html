Working Directory :
- The folder on your computer where project files exist and where you make changes (add, edit, delete).
● New files are not tracked until you (git add)them; changes to tracked files are automatically noticed by Git.

Staging Area :
- The middle step (a waiting room) where you put selected changes/files before committing.
● Controlled using git add.
● Helps you decide which modifications go into the next commit.

Repository (Repo) :
- The place where the code is stored with its full history (edits, revisions, commits, branches).
- project + all versions and their history + the different branches that run in a parallel timeline
● Local repository : on your pc.
● Remote repository : on a server (e.g., GitHub, GitLab, Bitbucket) accessible over the internet.

- Branch  
● a movable pointer to a commits
● saves and records new commits so you can develop independently without changing other branches.
● starts from an first commit
● Ends in Head (last commit)
● Creating a branch at Commit2 means it shares history with main up to Commit2, then each branch can evolve separately.
● So new-branch is like a sub-branch from main, not a totally independent project..

(main)
Commit1 --> Commit2 --> Commit3
              \
(new-branch)   NewCommit
