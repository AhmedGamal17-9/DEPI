form : 4 properties

action="" : بتحدد العنوان اللي هيتبعت له بيانات النموذج
method="" : بتحدد طريقة إرسال البيانات
method="get" : البيانات بتظهر في الرابط و مناسب للبحث والفلاتر.
method="post" : البيانات بتتبعت في جسم الطلب -البادي- و مناسب لتسجيل الدخول وإرسال معلومات حساسة
enctype="" :طريقة ترميز البيانات قبل الإرسال
novalidate : تمنع المتصفح من التحقق من البيانات قبل الإرسال

input : 15 properties

type="text" - type="password" - type="email" - type="number" - type="url" - type="tel"
type="search" : مربع بحث
type="date"- type="time" - type="month" - type="week"
type="datetime-local" : تاريخ + وقت (محلي)
type="button" - type="submit" - type="reset"
type="image" : زر ارسال سابميت ولكن في شكل صورة
type="color" - type="file"
type="checkbox" : مربع صغير  المستخدم يقدر يحدد أو يلغي التحديد (✅ / ⬜) و اختيار متعدد
type="radio" : دائرة صغير  المستخدم يقدر يحدد ويختار اختيار واحد بس من مجموعة
type="range" : اختيار رقم من شريط سحب
type="hidden" : حقل مخفي لإرسال بيانات للسيرفر
-----------------------
name="" : 
1-معرّف يتم إعطاؤه لحقل الإدخال بحيث يقدر السيرفر يتعرف على القيمة المدخلة لما يتم إرسال البيانات في الفورم
2- المتصفح بيجمع كل القيم الخاصة بالـ inputs.
3- ويربط كل قيمة ب النام الخاص بها.
4-ثم يرسلهم للسيرفر كـ key=value
مثال Username: Ahmed
5- في الراديو و الشيك بوكس لازم كلهم يبقا ليهم نفس النام
-----------------------
min="" - max="" :
الوصف: حدود دنيا وعليا لقيم رقمية أو تواريخ.
الأنواع المتأثرة: number, range, date, time, datetime-local, month, week.
-----------------------
value="" : قيمة ابتدائية للحقل لو اتسابت تُرسل عند الإرسال
placeholder="" : نص إرشادي يظهر داخل الحقل عندما يكون فارغًا
maxlength="" - minlength="" : أقل وأقصى عدد أحرف
step="" : خطوة الزيادة/النقصان للأرقام أو الأوقات
id="" : يُستخدم للربط مع label
required : يجعل الحقل مطلوبًا — المتصفح يمنع الإرسال قبل تعبئته
pattern="" : تتحقق من أن القيمة المدخلة من المستخدم تطابق شكل معين و بتشتغل مع الأنواع النصية وبيجي كود جاهز من مواقع
autofocus : يضع تركيز الماوس على الحقل عند تحميل الصفحة لأول مرة
autocomplete ="on": يسمح بالإكمال التلقائي
list="datalist-id" : بيربط الدات السيت بالانبت

readonly : 
1- الحقل للقراءة فقط بمعني المستخدم يقدر يشوف القيمة الموجودة فيه لكن ما يقدرش يعدلها
2- بتتبعت مع الفورم
3- بيشتغل مع input و textarea فقط
4- شكله عادي

disabled :
1- بتخلي عنصر الإدخال غير نشط و المستخدم ما يقدرش يتفاعل معاه
2- القيمة لا تُرسل مع الفورم
3- شغال مع معظم عناصر الفورم
4- شكله رمادي باهت
5- مفيد في الحالات التالية:
مثال استخدام :
- تعطيل زر "إرسال" لحد ما المستخدم يملأ البيانات المطلوبة
- تعطيل اختيارات معينة لحد ما الشرط يتحقق (مثلاً اختيار مدينة بعد اختيار الدولة)

Drag and Drop API (DnD API) : 
بيسمح إنك تسحب عنصر من مكان وتفلته في مكان تاني
------------------------------------------------------------------------------------------------------------------------------------------------------------------------
web storage API : 
بيوفر وسيلة لتخزين البيانات في المتصفح بشكل دائم أو مؤقت

localStorage → تخزين دائم حتى بعد إغلاق المتصفح.

sessionStorage → تخزين مؤقت (ينتهي مع غلق التبويب/المتصفح).
------------------------------------------------------------------------------------------------------------------------------------------------------------------------
web socket API : 
بروتوكول بيعمل اتصال دائم بين العميل والسيرفر في نفس الوقت - باي ديريكشن
 ------------------------------------------------------------------------------------------------------------------------------------------------------------------------
 web workers :
بيسمح بتشغيل أكواد JavaScript 
في خلفية المتصفح (background thread) بدون ما يوقف الصفحة الرئيسية.
